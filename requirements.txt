# FAISS — NVIDIA's CUDA-12 build, cuVS variant.
#
# cuVS unlocks the classic faiss-gpu IVFPQ M cap (which limited us to
# {1,2,3,4,8,12,16,20,24,28,32,40,48,56,64,96} regardless of GPU). With
# cuVS we can run M=256 PQ on GPU directly — no M=96 fallback needed,
# no CPU-build slowdown for high-recall indexes. Per FAISS wiki:
#
#   "The number of subquantizers representing a vector can be any
#    value less than or equal to the base dimension of the dataset...
#    GpuIndexIVFPQ indexes with 64 bytes per code or more do not
#    require the use of the float16 lookup tables for residual
#    distances."
#
# Trade-off: the cuvs build pulls RAFT + cuVS shared libs (~1-2 GB
# extra layer in the docker image, slower install). Worth it for the
# 10-20x build speedup on multi-million-vector M=256 IVFPQ indexes.
# Set FAISS_USE_CUVS=0 in env to fall back to classic kernels at
# runtime (still needs this package — switch is opt-out, not opt-in).
faiss-gpu-cuvs-cu12>=1.9.0

# safetensors retained for any consumer of MegaLoc weights at index time;
# torch / torchvision / timm come from the pytorch base image.
safetensors>=0.3.0
